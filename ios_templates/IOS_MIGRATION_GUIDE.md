# iOS Migration Guide — calcwise_core Portfolio

## Status

The 15 Flutter apps in the portfolio are **Android-only** in production. This guide
documents how to enable iOS for any one of them **without** disrupting the Android
build, store listing, or release cycle.

## Architecture target — separation of concerns

```
<AppName>/
├── android/                       ← Android-only, do NOT touch when activating iOS
│   ├── app/build.gradle.kts
│   ├── app/google-services.json   ← Android Firebase config (committed? check .gitignore)
│   └── app/src/main/AndroidManifest.xml
│
├── ios/                           ← iOS-only — created lazily, only touched by iOS work
│   ├── Runner/
│   │   ├── AppDelegate.swift           ← from ios_templates/
│   │   ├── Info.plist                  ← merged with ios_templates/Info.plist.additions.xml
│   │   ├── PrivacyInfo.xcprivacy       ← from ios_templates/
│   │   ├── SKAdNetworkItems.plist      ← from ios_templates/, merged into Info.plist
│   │   └── GoogleService-Info.plist    ← iOS Firebase config (DO NOT COMMIT)
│   ├── Podfile
│   └── Runner.xcworkspace
│
├── lib/                           ← Cross-platform Dart — platform branches use Platform.isIOS
│   └── core/ads/ad_config.dart    ← contains both bannerAndroid + banneriOS constants
│
└── pubspec.yaml                   ← shared, no per-platform sections needed
```

## What changes when iOS is activated for an app

| Layer            | File                                | Change                                                              |
|------------------|-------------------------------------|---------------------------------------------------------------------|
| Native iOS       | `ios/Runner/AppDelegate.swift`      | NEW — Firebase init + MobileAds + ATT                               |
| Native iOS       | `ios/Runner/Info.plist`             | MERGE — adds GADApplicationIdentifier, NSUserTracking…, SKAdNetwork |
| Native iOS       | `ios/Runner/PrivacyInfo.xcprivacy`  | NEW — Apple privacy manifest (iOS 17+ requirement)                  |
| Native iOS       | `ios/Runner/GoogleService-Info.plist` | NEW — Firebase iOS config (NEVER COMMIT)                          |
| Native iOS       | `ios/Podfile`                       | Generated by `flutter create --platforms=ios .`                     |
| Dart             | `lib/.../ad_config.dart`            | iOS ad unit ID constants — **already present** in most apps         |
| pubspec.yaml     | (no change)                         | No Dart packages are iOS-only in this stack                         |

## What stays untouched

When activating iOS for app X:

- **Nothing** under `<X>/android/`
- **Nothing** in `<X>/lib/` that doesn't already have a `Platform.isIOS` branch
- **Nothing** in `<X>/pubspec.yaml`
- **Nothing** in `<X>/test/`
- **Nothing** in any other portfolio app

## Files that must NEVER be touched in Android-only mode

If you're only working on Android:

- `ios_templates/` (this folder) — leave it alone
- Any `<App>/ios/` folder that exists from prior experiments — leave it alone
- The `Platform.isIOS` branches in `lib/` — they short-circuit on Android and add zero runtime cost

## Test impact

| Test type        | Re-run needed when iOS is activated? |
|------------------|---------------------------------------|
| Android unit     | NO — no Dart code changed                                            |
| Android widget   | NO — no UI code changed                                              |
| Android integration | NO                                                                |
| iOS unit         | YES — but these don't exist yet; create as part of iOS rollout       |
| iOS UI           | YES — manual smoke test on a real device + simulator                 |

**Bottom line**: activating iOS for any one app requires **zero re-test** of Android,
because no Android-impacting file is modified.

## Per-app iOS activation runbook

For each app (e.g. `MortgageUS`):

1. **Pre-flight**: Ensure ad_config.dart has `banneriOS`, `interstitialiOS`, `rewardediOS`
   constants (most apps already do; see audit notes).

2. **Scaffold iOS**:
   ```bash
   cd D:/mob/MortgageUS
   flutter create --platforms=ios .
   ```

3. **Dry-run the activator**:
   ```bash
   bash D:/mob/packages/calcwise_core/ios_templates/activate_ios.sh MortgageUS
   ```

4. **Apply**:
   ```bash
   bash D:/mob/packages/calcwise_core/ios_templates/activate_ios.sh MortgageUS --apply
   ```

5. **Manual TODOs** (printed by the script):
   - Add Firebase iOS app, drop in `GoogleService-Info.plist`
   - Create iOS AdMob app + units, fill into `ad_config.dart` + `Info.plist`
   - Merge `Info.plist.additions.xml` into the actual `Info.plist`
   - Merge `SKAdNetworkItems.plist` array under `SKAdNetworkItems` key
   - Add `PrivacyInfo.xcprivacy` to Xcode target
   - `cd ios && pod install`

6. **Build & smoke test**:
   ```bash
   flutter build ios --release --no-codesign
   ```

7. **Android regression check** (should be a no-op):
   ```bash
   flutter build apk --release
   flutter analyze --no-pub
   ```
